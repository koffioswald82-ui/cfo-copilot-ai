================================================================================
  1. QU'EST-CE QUE CFO COPILOT ?
================================================================================

CFO Copilot est un tableau de bord financier intelligent destiné aux directeurs
financiers (CFO) et aux analystes. Il permet de :

  - Charger des états financiers (compte de résultat, bilan, flux de trésorerie)
  - Calculer automatiquement les KPIs financiers clés
  - Détecter des anomalies et signaler les risques
  - Attribuer une note de santé financière (0 à 100)
  - Décomposer la rentabilité (analyse DuPont)
  - Simuler des scénarios économiques (récession, expansion...)
  - Prévoir les performances sur les 4 prochains trimestres
  - Comparer avec des benchmarks sectoriels
  - Poser des questions à un assistant IA (comme ChatGPT mais spécialisé CFO)

L'application est construite avec Python + Streamlit et déployée sur Streamlit
Cloud (accessible depuis n'importe quel navigateur, lien public pour le CV).

  3. LES DONNÉES D'ENTRÉE (CSV)
================================================================================

Le projet utilise 3 fichiers CSV correspondant aux 3 états financiers standards.

---- income_statement.csv (Compte de résultat) ----

Colonnes obligatoires :
  period                   → Trimestre au format "2023-Q1", "2023-Q2", etc.
  revenue                  → Chiffre d'affaires
  cost_of_goods_sold       → Coût des marchandises vendues (COGS)
  gross_profit             → Marge brute = revenue - COGS
  operating_expenses       → Charges d'exploitation (hors COGS)
  ebitda                   → Résultat avant intérêts, impôts, dépréciation
  depreciation_amortization → Dotation aux amortissements
  ebit                     → Résultat opérationnel = EBITDA - amortissements
  interest_expense         → Charges financières (intérêts de la dette)
  tax_expense              → Impôts sur les bénéfices
  net_income               → Bénéfice net

---- balance_sheet.csv (Bilan) ----

Colonnes obligatoires :
  period                   → Trimestre (même format)
  cash_and_equivalents     → Trésorerie
  accounts_receivable      → Créances clients
  inventory                → Stocks
  other_current_assets     → Autres actifs circulants
  total_current_assets     → Total actif circulant (somme des 4 ci-dessus)
  property_plant_equipment → Immobilisations corporelles (PPE)
  intangible_assets        → Immobilisations incorporelles (brevets, goodwill)
  other_noncurrent_assets  → Autres actifs non courants
  total_assets             → Total actif
  accounts_payable         → Dettes fournisseurs
  short_term_debt          → Dettes à court terme
  other_current_liabilities → Autres passifs courants
  total_current_liabilities → Total passif courant
  long_term_debt           → Dettes à long terme
  other_noncurrent_liabilities → Autres passifs non courants
  total_liabilities        → Total passif
  common_stock             → Capital social
  retained_earnings        → Réserves et bénéfices reportés
  total_equity             → Total capitaux propres

---- cash_flow.csv (Flux de trésorerie) ----

Colonnes obligatoires :
  period                   → Trimestre
  net_income               → Bénéfice net (point de départ)
  depreciation_amortization → Retraitement non-cash
  changes_in_working_capital → Variation du besoin en fonds de roulement
  other_operating          → Autres flux opérationnels
  operating_cash_flow      → Flux opérationnel total (le plus important)
  capital_expenditures     → Investissements (CAPEX)
  acquisitions             → Acquisitions d'entreprises
  other_investing          → Autres flux d'investissement
  investing_cash_flow      → Flux d'investissement total
  debt_issuance            → Nouvelles dettes levées
  debt_repayment           → Remboursement de dettes
  dividends_paid           → Dividendes versés
  other_financing          → Autres flux de financement
  financing_cash_flow      → Flux de financement total
  net_change_in_cash       → Variation nette de trésorerie

FORMAT DU TRIMESTRE :
  Le code attend le format "2023-Q1". Le loader le transforme en PeriodIndex
  ("2023Q1") qui est un index temporel trimestriel de Pandas.

  4. DATA_LOADER.PY — CHARGEMENT ET NETTOYAGE
================================================================================

Ce fichier a une seule mission : lire les CSV, vérifier qu'ils ont les bonnes
colonnes, nettoyer les données et retourner des DataFrames propres.

---- Étapes de traitement ----

ÉTAPE 1 : _parse_period()
  Transforme la colonne "period" (texte "2023-Q1") en PeriodIndex trimestriel.
  "2023-Q1" → remplace "-Q" par "Q" → "2023Q1" → pd.PeriodIndex(..., freq="Q")
  Le DataFrame est ensuite trié par ordre chronologique.

ÉTAPE 2 : _validate_schema()
  Vérifie que toutes les colonnes attendues sont présentes.
  Si une colonne manque → erreur explicite.

ÉTAPE 3 : _clean()
  - Convertit toutes les colonnes en nombres (pd.to_numeric)
  - Supprime les lignes entièrement vides
  - Remplit les valeurs manquantes isolées par interpolation linéaire
    (ex: si Q2 manque entre Q1=100 et Q3=120, il met Q2=110)

---- Fonctions disponibles ----

load_financial_data(data_dir)
  → Charge les 3 CSV depuis un dossier "data/"
  → Retourne un dict {"income_statement": df, "balance_sheet": df, "cash_flow": df}

load_from_dataframes(income_df, balance_df, cash_flow_df)
  → Même traitement mais à partir de DataFrames déjà en mémoire
  → Utilisé quand l'utilisateur upload ses propres fichiers dans l'app

get_combined_view(datasets)
  → Fusionne les 3 DataFrames en un seul (utile pour l'export ou l'IA)

  5. ANALYSIS.PY — LES MODÈLES FINANCIERS
================================================================================

C'est le cœur du projet. Ce fichier calcule tous les indicateurs financiers.

---- 5.1 LES KPIs CALCULÉS (FORMULES) ----

La fonction compute_kpis(income, balance, cash_flow) calcule :

1. CROISSANCE DU CHIFFRE D'AFFAIRES
   → QoQ (trimestre sur trimestre) = (CA[t] - CA[t-1]) / CA[t-1]
   → YoY (année sur année)        = (CA[t] - CA[t-4]) / CA[t-4]
   Code : rev.pct_change(1) et rev.pct_change(4)

2. MARGE BRUTE
   → gross_margin = gross_profit / revenue
   Exemple : si CA = 100 et marge brute = 40 → marge brute = 40%
   Interprétation : part du CA conservée après le coût de production.

3. MARGE EBITDA
   → ebitda_margin = ebitda / revenue
   L'EBITDA est le résultat AVANT intérêts, impôts, amortissements.
   C'est le meilleur indicateur de la rentabilité opérationnelle "pure".

4. MARGE NETTE
   → net_profit_margin = net_income / revenue
   Part du CA qui devient bénéfice réel (après tout).

5. MARGE DE CASH-FLOW OPÉRATIONNEL
   → operating_cash_flow_margin = operating_cash_flow / revenue
   Différence avec la marge nette : le cash-flow opérationnel mesure
   les vrais flux d'argent (pas les bénéfices comptables qui peuvent
   être artificiellement gonflés).

6. RATIO DE LIQUIDITÉ (Current Ratio)
   → current_ratio = total_current_assets / total_current_liabilities
   Règle : si > 1.0 → l'entreprise peut payer ses dettes à court terme
            si < 1.0 → risque de défaut à court terme (DANGER)
            si > 2.0 → très bonne liquidité

7. RATIO DETTE/CAPITAUX PROPRES (Debt-to-Equity)
   → total_debt = long_term_debt + short_term_debt
   → debt_to_equity = total_debt / total_equity
   Règle : si < 1.0 → peu endetté (solide)
            si > 2.0 → très endetté (risque financier)
            si > 3.0 → CRITIQUE

8. COUVERTURE DES INTÉRÊTS (Interest Coverage)
   → interest_coverage = ebit / interest_expense
   Combien de fois le résultat opérationnel couvre les intérêts.
   Règle : si < 1.5 → l'entreprise peine à payer ses intérêts
            si > 3.0 → bonne couverture

9. RETOUR SUR CAPITAUX PROPRES (ROE — Return on Equity)
   → avg_equity = moyenne mobile sur 2 périodes des capitaux propres
   → return_on_equity = net_income * 4 / avg_equity
   (multiplié par 4 pour annualiser un bénéfice trimestriel)
   Règle : si > 20% → excellente rentabilité pour les actionnaires

---- 5.2 DÉTECTION D'ANOMALIES (RÈGLES) ----

La fonction detect_anomalies() applique 5 règles de surveillance :

RÈGLE 1 — Chute de revenus
  Si revenue_growth_qoq < -15% → alerte "high"
  Si revenue_growth_qoq < -25% → alerte "critical"
  Seuil défini : REVENUE_DROP_THRESHOLD = -0.15

RÈGLE 2 — Explosion des coûts (COGS)
  Si croissance COGS > 20% ET croissance COGS > croissance CA + 5 points
  → alerte "high" (pression sur les marges)
  Seuil : COST_SURGE_THRESHOLD = 0.20
  Logique : les coûts augmentent plus vite que les revenus = marges en danger

RÈGLE 3 — Liquidité dangereuse
  Si current_ratio < 1.2 → alerte "high"
  Si current_ratio < 1.0 → alerte "critical"
  Seuil : CURRENT_RATIO_FLOOR = 1.2

RÈGLE 4 — Endettement excessif
  Si debt_to_equity > 2.0 → alerte "high"
  Si debt_to_equity > 3.0 → alerte "critical"
  Seuil : DEBT_EQUITY_CEILING = 2.0

RÈGLE 5 — Qualité des bénéfices (Earnings Quality)
  Si operating_cash_flow < net_income * 60% pendant 2 trimestres consécutifs
  → alerte "medium"
  Logique : si le bénéfice comptable est 40% supérieur au vrai cash généré,
  c'est suspect — peut indiquer de la manipulation comptable ou des créances
  non recouvrées.

SÉVÉRITÉ DES ALERTES (4 niveaux) :
  "low"      → information, surveiller
  "medium"   → attention requise
  "high"     → action recommandée
  "critical" → intervention immédiate nécessaire

---- 5.3 FINANCIAL HEALTH SCORE (NOTE 0–100) ----

La fonction compute_financial_health_score() calcule une note globale
composée de 5 dimensions. La note est calculée sur les DERNIÈRES VALEURS
disponibles (iloc[-1]).

DIMENSION 1 — RENTABILITÉ (max 25 points)
  Composantes :
    a) EBITDA margin :  si = 30% → 10 pts  (formule : min(ebitda_margin/0.30, 1.0) × 10)
    b) Marge nette    :  si = 15% →  8 pts  (formule : min(net_margin/0.15, 1.0) × 8)
    c) ROE            :  si = 20% →  7 pts  (formule : min(roe/0.20, 1.0) × 7)
  Total max : 25 pts

DIMENSION 2 — LIQUIDITÉ (max 20 points)
  Barème par paliers :
    current_ratio >= 2.0 → 20 pts (excellent)
    current_ratio >= 1.5 → 15 pts (bon)
    current_ratio >= 1.2 → 10 pts (acceptable)
    current_ratio >= 1.0 →  5 pts (fragile)
    current_ratio <  1.0 →  cr × 5 pts (dangereux)

DIMENSION 3 — LEVIER FINANCIER (max 20 points)
  Barème par paliers :
    debt_to_equity <= 0.5 → 20 pts (quasi pas de dette)
    debt_to_equity <= 1.0 → 16 pts (peu endetté)
    debt_to_equity <= 1.5 → 12 pts (modéré)
    debt_to_equity <= 2.0 →  8 pts (élevé)
    debt_to_equity >  2.0 →  max(0, 8 - (de-2.0)×3) pts (très endetté)

DIMENSION 4 — CROISSANCE (max 20 points)
  Composantes :
    a) Croissance QoQ : si = 5%  →  8 pts  (formule : min(qoq/0.05, 1.0) × 8)
    b) Croissance YoY : si = 15% → 12 pts  (formule : min(yoy/0.15, 1.0) × 12)

DIMENSION 5 — QUALITÉ DU CASH (max 15 points)
  OCF margin : si = 20% → 15 pts  (formule : min(ocf_margin/0.20, 1.0) × 15)

TOTAL = Dimension 1 + 2 + 3 + 4 + 5 (max = 100)

GRADES :
  ≥ 80 → Grade A — "Excellent financial health"           (vert)
  ≥ 65 → Grade B — "Good health — minor areas to monitor" (vert clair)
  ≥ 50 → Grade C — "Moderate — proactive management"      (orange)
  ≥ 35 → Grade D — "Weak — corrective action required"    (orange foncé)
  <  35 → Grade F — "Critical — immediate intervention"   (rouge)

---- 5.4 DÉCOMPOSITION DUPONT ----

La fonction compute_dupoint() décompose le ROE en 3 facteurs multiplicatifs.
C'est une analyse classique de la finance d'entreprise (créée par la société
DuPont dans les années 1920).

FORMULE :
  ROE = Marge Nette × Rotation des Actifs × Multiplicateur des Capitaux Propres

En détail :
  Net Profit Margin    = net_income / revenue
                         (combien de bénéfice par dollar de CA)

  Asset Turnover       = revenue / total_assets
                         (efficacité : combien de CA généré par dollar d'actif)

  Equity Multiplier    = total_assets / total_equity
                         (levier : combien d'actifs pour 1 dollar de capital propre)

  ROE (DuPont)         = Net Margin × Asset Turnover × Equity Multiplier × 4
                         (× 4 pour annualiser depuis un trimestre)

INTERPRÉTATION :
  Si le ROE est faible, DuPont permet de diagnostiquer POURQUOI :
  - Marge faible → problème de rentabilité opérationnelle
  - Rotation faible → actifs sous-utilisés (trop d'inventaire, etc.)
  - Multiplicateur faible → sous-levier (peut être une opportunité)

---- 5.5 ANALYSE DE SCÉNARIOS (STRESS TEST) ----

La fonction run_scenario_analysis() applique des chocs sur le DERNIER trimestre
et recalcule le compte de résultat entier.

PARAMÈTRES D'ENTRÉE :
  revenue_delta  : variation du CA en fraction (ex: -0.15 = -15%)
  cogs_pp_delta  : variation du ratio COGS/CA en points de % (ex: 0.05 = +5pp)
  opex_delta     : variation des charges opex en fraction (ex: 0.10 = +10%)
  interest_delta : variation des charges financières (ex: 0.50 = +50%)

CALCULS EFFECTUÉS (dans l'ordre) :
  1. nouveau_CA     = CA_base × (1 + revenue_delta)
  2. nouveau_COGS   = nouveau_CA × (ratio_COGS_base + cogs_pp_delta)
  3. nouvelle_marge_brute = nouveau_CA - nouveau_COGS
  4. nouveau_OPEX   = OPEX_base × (1 + opex_delta)
  5. nouvel_EBITDA  = nouvelle_marge_brute - nouveau_OPEX
  6. nouvel_EBIT    = nouvel_EBITDA - amortissements_base
  7. nouveaux_intérêts = intérêts_base × (1 + interest_delta)
  8. EBT            = nouvel_EBIT - nouveaux_intérêts
  9. nouvel_impôt   = max(0, EBT × taux_impôt_base)
  10. nouveau_résultat_net = EBT - nouvel_impôt

Le taux d'impôt est calculé dynamiquement depuis les données historiques :
  taux_impôt = tax_expense / (ebit - interest_expense)

SCÉNARIOS PRÉDÉFINIS (6 scénarios) :
  "Base Case"   : aucun choc (0%, 0%, 0%, 0%)
  "Recession"   : CA -15%, COGS +3pp, Opex +5%, Intérêts +50%
  "Expansion"   : CA +20%, COGS -1pp, Opex +10%, Intérêts 0%
  "Cost Crisis" : CA 0%,   COGS +8pp, Opex +20%, Intérêts +100%
  "Best Case"   : CA +25%, COGS -2pp, Opex -5%, Intérêts -30%
  "Stagflation" : CA +3%,  COGS +5pp, Opex +10%, Intérêts +150%

SÉVÉRITÉ DU RÉSULTAT :
  impact EBITDA >= 10%  → "positive" (bonne nouvelle)
  impact EBITDA >= -5%  → "neutral"
  impact EBITDA >= -20% → "warning"
  impact EBITDA <  -20% → "critical"

BENCHMARKS SECTORIELS (7 secteurs disponibles) :
  Technology       : marge brute 65%, EBITDA 25%, D/E 0.5, ROE 20%
  Industrials      : marge brute 35%, EBITDA 15%, D/E 0.8, ROE 12%
  Consumer Goods   : marge brute 45%, EBITDA 18%, D/E 1.0, ROE 14%
  Healthcare       : marge brute 55%, EBITDA 22%, D/E 0.6, ROE 16%
  Financial Services: marge brute 70%, EBITDA 30%, D/E 1.5, ROE 15%
  Retail           : marge brute 40%, EBITDA 12%, D/E 1.2, ROE 10%
  Energy           : marge brute 30%, EBITDA 20%, D/E 0.7, ROE 11%

  Ces benchmarks servent à comparer les KPIs de l'entreprise analysée
  avec les moyennes de son secteur (affiché en graphique radar).

  6. FORECASTING.PY — PRÉVISIONS FINANCIÈRES
================================================================================

Ce fichier prédit les 4 prochains trimestres pour 4 métriques :
  - Revenue (chiffre d'affaires)
  - EBITDA
  - Net Income (bénéfice net)
  - Operating Cash Flow

Deux modèles sont disponibles. Le système choisit automatiquement le meilleur.

---- 6.1 MODÈLE ARIMA ----

ARIMA = AutoRegressive Integrated Moving Average.
C'est le modèle classique de prévision de séries temporelles en finance.

Configuration utilisée : ARIMA(1,1,1)(1,1,0)[4]
  - (1,1,1) → partie non-saisonnière
      1 = 1 terme autorégressif (utilise t-1 pour prédire t)
      1 = 1 différenciation (pour rendre la série stationnaire)
      1 = 1 terme de moyenne mobile (corrige les erreurs passées)
  - (1,1,0)[4] → partie saisonnière (cycle de 4 trimestres = 1 an)
      1 = 1 terme autorégressif saisonnier
      1 = 1 différenciation saisonnière
      0 = pas de terme MA saisonnier

PROCESSUS :
  1. Séparer : 80% des données pour entraîner, 20% (4 derniers) pour tester
  2. Entraîner ARIMA sur les 80%
  3. Calculer le MAPE (erreur %) sur les 20% (mesure de fiabilité)
  4. Réentraîner sur 100% des données
  5. Générer les prévisions avec intervalles de confiance à 80%

INTERVALLE DE CONFIANCE 80% :
  Le modèle donne 3 valeurs pour chaque trimestre prédit :
    - Prévision centrale (la plus probable)
    - Borne basse (20% de chances que ce soit plus bas)
    - Borne haute (20% de chances que ce soit plus haut)
  → L'espace entre borne basse et borne haute est la "zone d'incertitude"

---- 6.2 RÉGRESSION RIDGE ----

Ridge Regression est un modèle de machine learning (régression linéaire
régularisée). Il est utilisé en "mode autorégressif" : il prédit le prochain
trimestre à partir des 4 trimestres précédents.

PROCESSUS :
  1. Construire la matrice de variables (X) : chaque ligne = 4 trimestres
     précédents, chaque cible (y) = trimestre suivant
  2. Normaliser les données (StandardScaler : moyenne=0, écart-type=1)
  3. Entraîner Ridge avec alpha=1.0 (paramètre de régularisation)
  4. Pour la prévision : prédire t+1, puis utiliser cette prévision pour
     prédire t+2, etc. (méthode "rolling window")
  5. Calculer l'incertitude qui grandit avec l'horizon :
     marge = 1.28 × écart-type_résidus × √(étape)
     (1.28 = z-score à 80% de confiance)

PARAMÈTRE alpha=1.0 :
  La régularisation Ridge évite le surapprentissage (overfitting).
  Plus alpha est grand, plus le modèle est "prudent" et évite les
  prévisions extrêmes.

---- 6.3 CHOIX AUTOMATIQUE DU MODÈLE ----

La fonction forecast_metric() avec strategy="auto" :
  1. Essaie ARIMA
  2. Si ARIMA échoue OU si son MAPE dépasse 50% → utilise Ridge
  3. Sinon → garde ARIMA

MAPE = Mean Absolute Percentage Error = erreur moyenne en pourcentage.
  Exemple : MAPE = 5% signifie que les prévisions se trompent de 5% en moyenne.
  Interprétation :
    < 5%  → Excellent
    < 15% → Bon
    > 15% → Acceptable (les données financières sont parfois imprévisibles)

MINIMUM REQUIS : 8 trimestres de données historiques pour pouvoir prévoir.

  7. LLM_ASSISTANT.PY — L'INTELLIGENCE ARTIFICIELLE

Ce fichier intègre un assistant IA spécialisé en finance.
"LLM" = Large Language Model = modèle de langage (comme ChatGPT).

---- 7.1 COMMENT ÇA MARCHE ----

PRINCIPE :
  1. On prend les KPIs, le score de santé, les anomalies et les prévisions
  2. On les formate en texte structuré (le "contexte financier")
  3. On envoie ce texte au modèle IA avec une question
  4. Le modèle retourne une réponse rédigée en langage naturel

PROMPT SYSTÈME (instructions données à l'IA) :
  L'IA est configurée comme "Expert CFO consultant niveau Partner Big 4".
  Règles imposées à l'IA :
    - Être concis et factuel (citer les chiffres exacts)
    - Signaler les risques proactivement
    - Structurer : situation → problème → recommandation
    - Ne jamais inventer de chiffres (que les données fournies)
    - Utiliser le jargon financier mais l'expliquer si nécessaire

---- 7.2 LES FOURNISSEURS D'IA GRATUITS ----

TOUS les fournisseurs gratuits utilisent la MÊME interface technique (OpenAI SDK)
mais avec une URL de base différente. C'est le génie de l'approche "OpenAI-compat".

---- 7.3 SYSTÈME ANTI-BLOCAGE (RETRY + FALLBACK) ----

Problème : les APIs gratuites ont des limites de requêtes (429 = trop de requêtes).

SOLUTION EN 2 NIVEAUX :

Niveau 1 — Retry avec attente progressive (dans chaque backend) :
  Tentative 1 → si 429 → attendre 30 secondes
  Tentative 2 → si 429 → attendre 60 secondes
  Tentative 3 → si 429 → lever une exception

Niveau 2 — Fallback automatique (dans CFOAssistant) :
  Si le fournisseur principal échoue → essayer automatiquement dans l'ordre :
    1. Groq → 2. Together AI → 3. OpenRouter → 4. Mistral → 5. Ollama
  Ne passe au suivant que si le précédent n'a pas de clé ou échoue.

---- 7.4 LES 4 FONCTIONS IA ----

1. generate_executive_summary(financial_data)
   Génère un rapport exécutif complet en 8 sections :
   executive summary, performance, cash-flow, dette, score santé,
   risques, prévisions, recommandations stratégiques.
   → max_tokens=1800 (réponse longue)

2. generate_scenario_narrative(scenario_name, base, scenario, assumptions)
   Rédige 4-6 phrases expliquant un scénario de stress :
   représentation stratégique, sévérité, risque principal, mitigation.
   → max_tokens=500

3. explain_kpi_variance(metric, current, prior, financial_data)
   Explique en 3-4 phrases pourquoi un KPI a changé et si c'est inquiétant.
   → max_tokens=500

4. answer_question(question, financial_data)
   Répond à des questions libres (chat). Maintient l'historique de la
   conversation pour que l'IA se souvienne des échanges précédents.
   → max_tokens=800

---- 8.2 LES GRAPHIQUES (PLOTLY) ----

Le projet utilise la bibliothèque Plotly (graphiques interactifs).

  go.Figure()          → crée un graphique vide
  fig.add_trace(...)   → ajoute une série de données
  fig.add_vline(...)   → ajoute une ligne verticale (fonctionne sur axes numériques)
  fig.add_shape(...)   → ajoute une forme géométrique
  fig.add_annotation() → ajoute un label texte sur le graphique
  st.plotly_chart(fig) → affiche le graphique dans Streamlit

  go.Scatter → graphique linéaire
  go.Bar     → graphique en barres
  go.Waterfall → graphique cascade (utilisé pour l'impact scénario)
  go.Indicator → jauge, KPI affiché en grand
  go.Scatterpolar → graphique radar (comparaison sectorielle)

---- 8.3 UPLOAD DE DONNÉES RÉELLES ----

FONCTIONNEMENT :
  1. L'utilisateur uploade un fichier Excel ou 3 CSV
  2. Les données sont traitées par load_from_dataframes()
  3. Stockées dans st.session_state["uploaded_datasets"]
  4. La fonction main() vérifie en premier ce session_state
  5. Si données uploadées → toutes les analyses utilisent ces données
  6. Sinon → utilise les données exemple (data/)

SESSION STATE :
  Streamlit est sans état (stateless) par défaut : chaque interaction recharge
  la page. st.session_state est un dictionnaire persistant pendant la session.
  C'est le seul moyen de "garder" des données entre les interactions.

================================================================================
  RÉSUMÉ DES FORMULES FINANCIÈRES CLÉS
================================================================================

  Marge brute          = (CA - COGS) / CA
  Marge EBITDA         = EBITDA / CA
  Marge nette          = Bénéfice net / CA
  Croissance QoQ       = (CA[t] - CA[t-1]) / CA[t-1]
  Croissance YoY       = (CA[t] - CA[t-4]) / CA[t-4]
  Current Ratio        = Actif courant / Passif courant
  Debt-to-Equity       = (LT dette + CT dette) / Capitaux propres
  Interest Coverage    = EBIT / Charges financières
  ROE                  = Bénéfice net × 4 / Capitaux propres moyens
  ROE (DuPont)         = Marge nette × (CA/Actif total) × (Actif total/CP)
  OCF Margin           = Cash-flow opérationnel / CA
